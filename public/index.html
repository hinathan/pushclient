CATT Client demonstrator

This simple website shows the response performance and demonstrates the output from the Convert All The Things technology preview. It uses the standard API to initiate a conversion and acts as the input source and output sink handler for the API. This is necessary since the CATT API is inherently asynchronous — it's not 100% assured that a conversion will complete immediately so the API takes a destination URL as a required argument. Once our system has completed the requested document transform, we POST the results to the URL your client specified.

What's happening?

Some preview elements can be readily extracted with our own custom toolchain. However, a full-quality PDF which retains all of the formatting of the original document requires using the native applications on their native platform. In most cases we can quickly generate a small thumbnail of the document. The full PDF takes a few seconds more.
